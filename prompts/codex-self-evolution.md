# Codex 自我进化提示词

这个提示词适合定期让 Codex 回看近期工作，把重复出现、耗时、容易出错、上下文负担重的任务沉淀成可复用资产。

它不要求 Codex 盲目创建一堆 skills，而是先做证据分析，再决定应该沉淀成 skill、subagent、automation，还是暂时跳过。

## 适合什么时候用

- 最近连续做了很多类似任务，想总结成稳定工作流。
- 感觉 Codex 每次都在重复理解同一套上下文。
- 想找出哪些任务值得做成 skills。
- 想清理已有 skills、agents、automations 是否重复。
- 想做一次个人工作流复盘。

## 通用模板

```text
回看我最近 30 天的工作记录；如果不足 30 天，就使用所有可用历史。

请按以下证据顺序分析：

1. 最近的 Codex 会话、任务总结和产出记录。
2. Codex Memories、rollout summaries 或类似长期记忆，用来发现跨会话重复出现的模式。
3. 如果启用了外部工作记录系统，可以用它发现 Codex 之外的重复工作；但重要细节必须回到原始系统确认。
4. 已有的 skills、custom agents、automations 和项目规则，优先复用或扩展已有资产，避免重复创建。

请广泛寻找那些重复、耗时、容易出错、上下文负担重，或者明显受益于稳定流程的工作。范围可以包括：写代码、研究、写作、规划、沟通、运营、分析、个人事务等。

只有当一个候选工作满足以下条件时，才考虑处理：

- 至少出现过两次，或者明显会再次发生且重复成本较高。
- 有稳定输入、可重复步骤、明确输出或停止条件。
- 能明显提升速度、质量、一致性或可靠性。
- 没有被现有 skill、agent 或 automation 充分覆盖。

请为每个候选选择最小合适形式：

- Skill：可复用的工作流或操作手册。
- Custom subagent：边界清晰、适合委派的专家角色或调查任务。
- Automation：定期检查、报告、提醒或监控。
- Extend existing：扩展已有 skill、agent 或 automation。
- Skip：一次性、模糊、敏感、证据不足，或不值得沉淀的工作。

先输出一个紧凑 shortlist，包含：

- 重复工作流
- 支持证据和日期
- 频率 / 置信度
- 推荐形式
- 为什么值得或不值得创建
- 隐私或安全风险

然后只创建或修改高置信、确实缺失的项目。

创建时保持范围窄、实用、基于证据、容易验证。不要创建投机性的、重复的、过宽的资产。

最后总结：

- 你创建或扩展了什么
- 你刻意跳过了什么
- 哪些还需要更多证据才能沉淀
```

## 使用建议

- 不要每天都跑，建议每 1-2 周或一段密集项目后使用。
- 如果输出里出现敏感项目，先只做本地清单，不要直接发布。
- 不要让 Codex 自动公开历史记录；公开前单独做脱敏和授权检查。
- 如果一个 workflow 只是出现过一次，不要急着做成 skill。

## 判断标准

一个好 skill 通常应该满足：

- 输入稳定。
- 步骤可复用。
- 输出可验证。
- 失败边界清楚。
- 未来会再次出现。

不适合沉淀的内容：

- 一次性任务。
- 强个人隐私任务。
- 依赖临时账号、密钥、服务器状态的任务。
- 只是某个项目里的细碎事实。
- 已被现有 skill 充分覆盖的工作。
