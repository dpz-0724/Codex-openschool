# AI 教程视觉提示词方法：参考宝玉 skills

这份方法论整理自本地对宝玉 skills 的学习和使用经验。公开分享时只保留通用视觉路线，不复制外部 skill 的完整模板。

来源参考：

- 宝玉 skills：`https://github.com/JimLiu/baoyu-skills`

## 核心原则

做 AI 教程类视觉时，不要只追求“科技感”。更重要的是让观众一眼看懂结构、风险、流程和证据。

推荐把视觉分成四层：

1. 封面钩子：先让人知道这条内容在解决什么问题。
2. 概念解释：用结构图、箭头、卡片解释复杂概念。
3. 证据窗口：用截图、小窗、标注提供可信依据。
4. 总结卡片：用口诀、清单、风险提醒帮观众记住。

## 推荐视觉路线

### 1. 手绘知识卡片

适合解释：

- AI Agent 工作流
- Codex / OpenClaw / Playwright
- API、浏览器、桌面软件、视觉兜底
- 多工具协作和流程拆解

视觉关键词：

- 温暖纸张背景
- 柔和色块
- 手绘线条
- 圆角信息区
- 波浪箭头
- 浏览器、终端、API、齿轮、清单等 doodle 图标
- 充足留白

避免：

- 过度科幻
- 蓝色抽象数据海洋
- 没有信息结构的装饰图

### 2. 证据截图小窗

适合解释：

- 官方文档
- API 返回
- 工具页面
- 浏览器调试流程

做法：

- 不让截图占满画面。
- 把截图作为证据卡片嵌入知识图里。
- 用框选、箭头、放大和高亮说明重点。
- 保留可信依据，但不让截图破坏整体风格。

### 3. 丝网印刷风封面

适合：

- 强观点
- 风险提醒
- 前 3 秒钩子
- 标题冲击

视觉关键词：

- 平面色块
- 少量高对比色
- 粗形状和象征符号
- 半调网点
- 字体和画面融在一起

### 4. 真实学习笔记感

适合：

- 总结页
- 避坑清单
- 操作口诀
- 复盘卡片

视觉关键词：

- 俯拍笔记纸
- 蓝色圆珠笔字
- 红笔圈注
- 黄色荧光笔
- 边注和订正痕迹

## 一条 AI 教程视频的视觉组合

可以按这个顺序组织：

1. 封面：用强钩子说明问题。
2. 第一段：用手绘结构图解释“这件事到底分几层”。
3. 第二段：用截图小窗证明工具或文档依据。
4. 第三段：用卡片动画说明操作顺序。
5. 结尾：用学习笔记风总结口诀和风险点。

## 禁止路线

- 不要用空泛的 AI 大脑、机械臂、蓝色宇宙、抽象粒子当主视觉。
- 不要用办公室摆拍假装真实工作流。
- 不要让画面只靠字幕变化。
- 不要先生成空白图，再用后期强行覆盖核心文字。
- 不要把长段正文塞进图片。

## 一个可复用的提示词骨架

```text
生成一张 AI 教程风格的知识卡片。

主题：[写清楚这张图解释什么]
目标观众：[新手 / 开发者 / 运营 / 产品经理]
画面结构：[封面钩子 / 流程图 / 对比图 / 总结卡片]
核心元素：[浏览器窗口、终端、API、清单、箭头、风险标记等]
文字要求：[只放短标题、短标签、口诀，不要长段正文]
风格要求：[手绘知识卡片 / 丝网印刷封面 / 真实学习笔记]
颜色要求：[柔和纸感 / 高对比平面色 / 红蓝笔记感]
禁止内容：[不要科幻大脑、不要机械臂、不要办公室摆拍、不要抽象粒子]
```

## 发布提示

这类提示词最适合配合真实截图、真实操作记录和清晰脚本使用。图像不是装饰，而是帮观众理解复杂技术流程。
