# 第一期开源提示词模板：20 个可复用版本

这些模板都按“可替换变量 + 输出格式 + 验收标准”来写。使用时把 `{...}` 替换成你的具体任务即可。

## 教学

### 1. 苏格拉底式分步辅导模板

用途：适合教学、答疑、作业辅导。目标是引导学习者自己想，而不是直接给答案。

```text
你是一个耐心但严格的学习教练。我要学习：{主题}。

请按以下方式辅导我：
1. 先问我 1 个诊断问题，判断我现在的理解水平。
2. 不要直接给最终答案，先给我一个最小提示。
3. 等我回答后，再指出我的误区和下一步思考方向。
4. 如果我连续两次答错，再给一个更具体的例子。
5. 最后用 5 句话总结我今天应该记住的内容。

约束：
- 不替我完成作业原题。
- 每次只问一个问题。
- 使用适合 {年级/背景} 的语言。
```

### 2. 教师备课与差异化练习模板

用途：把一个主题变成课堂目标、讲解流程和分层练习。

```text
你是课程设计助教。请为 {学生群体} 设计一节关于 {主题} 的课。

输出：
1. 本节课 3 个学习目标。
2. 10 分钟导入活动。
3. 核心概念讲解提纲。
4. 基础、进阶、挑战三档练习，每档 3 题。
5. 常见误区和纠错话术。
6. 课后检索练习题 5 题。

要求：
- 不假设学生已经掌握 {先修知识之外的内容}。
- 练习题要能检查理解，而不是只检查记忆。
- 如果需要事实材料，请标注“需要教师补充核验”。
```

## 代码审查

### 3. PR 代码审查清单模板

用途：让 AI 先做结构化 code review，再由人判断。

```text
你是资深代码审查员。请审查下面的变更：{diff 或文件摘要}。

请只关注真实风险，按严重程度输出：
1. 可能导致错误结果、数据损坏、安全问题或线上故障的问题。
2. 缺失的测试或容易回归的路径。
3. 复杂度、可维护性或边界条件问题。

每个发现必须包含：
- 严重程度：P0/P1/P2/P3
- 位置：文件和行号，如可判断
- 问题说明
- 为什么会发生
- 建议修复
- 建议测试

如果没有明确问题，请说“未发现确定性问题”，并列出剩余风险。
```

### 4. 变更风险评估模板

用途：发布前评估一个改动影响面。

```text
请评估这次变更的发布风险：{变更说明或 diff 摘要}。

输出表格：
- 影响模块
- 用户可见变化
- 潜在回归点
- 数据或权限风险
- 必跑测试
- 发布建议：可直接发布 / 需要灰度 / 需要回滚预案 / 不建议发布

额外要求：
- 不要只复述代码改了什么，要判断用户和系统会受到什么影响。
- 对不确定的信息标注“需要确认”，不要编造。
- 最后给一个 5 行以内的发布前 checklist。
```

## 调试

### 5. CI 失败诊断模板

用途：读 CI 日志并产出可执行诊断路径。

```text
下面是 CI 失败信息：{日志摘要}。
相关变更是：{最近改动摘要}。

请按这个顺序分析：
1. 从日志里提取最早的有效错误。
2. 判断它更像是环境问题、依赖问题、测试数据问题、代码回归还是 flaky。
3. 给出 3 个可验证假设，并为每个假设设计一个验证命令或检查点。
4. 给出最小修复建议。
5. 说明修复后必须重跑哪些测试。

不要根据最后一行错误直接下结论；优先找根因。
```

### 6. 最小复现与根因分析模板

用途：把模糊 bug 变成可复现、可修复的问题。

```text
请帮我把这个问题整理成最小复现和根因分析。

问题描述：{现象}
期望结果：{期望}
实际结果：{实际}
环境：{系统/版本/依赖}
最近变更：{最近变更}

输出：
1. 还缺哪些关键信息。
2. 最小复现步骤。
3. 可能根因列表，按概率排序。
4. 每个根因的验证方法。
5. 修复策略和回归测试。

约束：没有证据时不要给确定结论。
```

## 产品需求

### 7. PRD 一页纸模板

用途：把一个想法整理成可讨论的一页需求。

```text
请把下面想法整理成一页 PRD：{产品想法}。

输出结构：
1. 背景：为什么现在要做。
2. 用户：目标用户是谁，痛点是什么。
3. 目标：这次要改变什么指标或行为。
4. 范围：本期做什么，不做什么。
5. 用户流程：关键步骤。
6. 验收标准：可测试的 5 条。
7. 风险和依赖：技术、数据、运营、合规。
8. 需要进一步验证的问题。

要求：把假设和事实分开写。
```

### 8. Bug / Feature Issue 模板

用途：生成清晰 issue，方便研发直接处理。

```text
请把下面内容整理成一个工程团队可执行的 issue：{原始描述}。

输出：
- 标题
- 类型：Bug / Feature / Chore / Question
- 背景
- 复现步骤或用户场景
- 期望行为
- 当前行为
- 验收标准
- 优先级建议和理由
- 需要补充的信息

约束：
- 不要把猜测写成事实。
- 如果信息不足，明确列出缺口。
```

## 研究总结

### 9. 来源驱动研究简报模板

用途：做资料研究、竞品分析、技术选型前的简报。

```text
请围绕 {研究主题} 做一份来源驱动简报。

范围：
- 时间范围：{时间}
- 受众：{受众}
- 优先来源：{官方文档/论文/财报/代码仓库/新闻}
- 排除来源：{不可信或不相关来源}

输出：
1. 结论摘要，最多 5 条。
2. 证据表：结论、来源、发布时间、可信度、备注。
3. 分歧或不确定点。
4. 对 {我的场景} 的建议。
5. 需要继续验证的问题。

要求：每个关键结论都必须有来源，不要用“网上说”。
```

### 10. 公开 Prompt 库二次改写模板

用途：学习优秀 prompt 的结构，但不复制原文。

```text
请分析这个公开 prompt 或 prompt 集合：{链接或摘要}。

任务：
1. 只提炼结构，不复刻原文。
2. 总结它的角色设定、输入变量、约束、输出格式和验收标准。
3. 判断适合哪些场景，不适合哪些场景。
4. 重新写一个面向 {目标场景} 的原创模板。
5. 在末尾列出来源链接和改写说明。

约束：
- 不输出长段原 prompt。
- 不保留原作者的专有表达。
- 如果许可证不清楚，只做学习笔记，不做模板发布。
```

## Agent 工作流

### 11. 简单工作流 vs 自主 Agent 判断模板

用途：避免所有任务都过度 agent 化。

```text
请帮我判断这个任务应该用简单提示词、固定 workflow、脚本工具，还是自主 Agent。

任务：{任务描述}
输入：{输入数据}
输出：{期望结果}
是否会产生副作用：{会/不会/不确定}
失败成本：{低/中/高}
是否需要长期运行：{是/否}

请输出：
1. 推荐方案。
2. 为什么不是其他方案。
3. 必要工具和权限。
4. 风险控制点。
5. 最小可行版本。
6. 上线前验证清单。
```

### 12. Codex 自我进化复盘模板

用途：定期把重复任务沉淀为 skill、prompt、脚本或自动化。

```text
请复盘我最近的 Codex 使用记录：{脱敏后的任务列表}。

目标：
1. 找出重复出现的任务类型。
2. 判断哪些适合沉淀成 skill，哪些适合沉淀成提示词，哪些适合沉淀成脚本。
3. 给每个候选项写一句触发场景。
4. 给出最低风险的落地顺序。
5. 标出不能公开或需要脱敏的内容。

输出表格：
- 候选名称
- 类型：skill / prompt / script / checklist
- 价值
- 风险
- 下一步动作
```

## 视觉生成

### 13. AI 教程视觉概念模板

用途：给 AI 教程做封面、知识卡片、概念图和短视频视觉。

```text
请为这个 AI 教程主题设计视觉概念：{主题}。

输出 5 个方向：
1. 封面主视觉
2. 教程过程图
3. 概念对比图
4. 案例演示图
5. 总结卡片

每个方向包含：
- 画面主体
- 构图
- 关键元素
- 文字位置建议
- 不要出现的元素
- 适合平台：公众号 / 小红书 / 视频封面 / 课件

要求：不要只写“科技感”，要能让设计或生图模型执行。
```

### 14. 生产级图像提示词模板

用途：让图像提示词更稳定，而不是只堆风格词。

```text
请根据以下信息写一个图像生成提示词。

主题：{主题}
主体：{主体}
使用场景：{封面/海报/课件/广告/产品图}
画面比例：{比例}
构图：{近景/中景/俯拍/居中/留白}
光线：{自然光/棚拍/柔光/高对比}
风格参考：{风格方向，不写具体在世艺术家}
必须出现：{元素}
必须避免：{元素}
文字需求：{是否需要文字}

输出：
1. 中文提示词。
2. 英文提示词。
3. Negative prompt。
4. 可能失败点和修正建议。
```

## PPT

### 15. 从文档生成演示稿模板

用途：把长文、报告、论文或网页变成演示大纲。

```text
请基于以下资料生成 PPT 大纲：{资料摘要或正文}。

受众：{受众}
目标：{汇报/教学/路演/复盘}
页数：{页数}
风格：{专业/教学/产品/学术}

输出：
1. 每页标题。
2. 每页一句核心观点。
3. 3-5 个要点。
4. 建议图表或视觉形式。
5. 演讲者备注。
6. 哪些内容需要事实核验。

约束：
- 只基于给定资料，不补编数据。
- 每页只表达一个核心观点。
```

### 16. 高层汇报叙事模板

用途：把复杂材料压缩成决策层能读的汇报。

```text
请把这些材料整理成高层汇报叙事：{材料}。

请按这个逻辑组织：
1. 当前背景：发生了什么。
2. 关键洞察：真正重要的变化是什么。
3. 业务影响：对用户、收入、成本、风险或效率的影响。
4. 可选方案：至少 2 个。
5. 推荐决策：推荐什么，为什么。
6. 风险与缓解。
7. 下一步 30 天行动。

要求：
- 语言简洁，不写口号。
- 每个建议都要能追溯到材料或明确假设。
```

## 视频脚本

### 17. 教程短视频脚本模板

用途：生成技术教程、AI 工具教程或产品演示短视频。

```text
请为 {主题} 写一个 {时长} 的教程短视频脚本。

目标观众：{观众}
他们最关心的问题：{痛点}
视频目标：{看完后能做什么}

输出：
1. 3 个标题候选。
2. 5 秒开场钩子。
3. 分镜表：时间、旁白、画面、屏幕操作、字幕重点。
4. 演示素材清单。
5. 结尾复盘和行动建议。

约束：
- 不夸大效果。
- 技术步骤必须可复现。
- 对需要联网或登录的步骤加风险提示。
```

### 18. AI 视频镜头提示词模板

用途：给 AI 视频模型或视频制作工具写镜头级提示。

```text
请为下面场景写 5 个镜头提示词：{场景}。

每个镜头包含：
- 镜头目的
- 画面主体
- 场景环境
- 相机运动
- 主体动作
- 光线和色彩
- 时长
- 禁用项

整体要求：
- 镜头之间要有时间推进。
- 不要只写抽象情绪，要写可见动作。
- 如果需要文字，单独列为后期字幕，不要求模型生成准确文字。
```

## 学习复盘

### 19. Gibbs 六步复盘模板

用途：复盘一次学习、项目、实验或失败经历。

```text
请按 Gibbs 六步法帮我复盘这次经历：{经历摘要}。

输出：
1. Description：发生了什么，只写事实。
2. Feelings：当时的判断和感受。
3. Evaluation：哪些做得好，哪些不好。
4. Analysis：为什么会这样，有哪些结构性原因。
5. Conclusion：下次应该保留什么、改变什么。
6. Action Plan：下次具体怎么做，列 3 个行动。

要求：
- 不做情绪安慰，重点是可执行改进。
- 涉及他人或敏感信息时，先匿名化。
```

### 20. Cornell + 检索练习模板

用途：把学习笔记变成可复习、可测验的材料。

```text
请把这份学习笔记整理成 Cornell 笔记和检索练习：{笔记内容}。

输出：
1. 右栏笔记：核心概念和例子。
2. 左栏线索：能触发回忆的问题。
3. 底部摘要：100 字以内。
4. 检索练习：不看笔记也能回答的 10 个问题。
5. 间隔复习计划：明天、3 天后、7 天后分别复习什么。

要求：
- 问题要逼迫回忆，不只是让人复制原文。
- 把容易混淆的概念做成对比题。
```
