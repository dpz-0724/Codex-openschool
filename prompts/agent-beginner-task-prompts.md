# Agent 新手任务提示词包：7 天完成第一个可验证闭环

## 审计信息

- 来源：本地 Codex / skills / 自动化浏览 / 知识库维护经验；结合 OpenAI prompt guidance、OpenAI Agents SDK、Anthropic prompt engineering 文档整理。
- 作者：本地整理；公开版由 Codex 脱敏改写。
- 适合读者：已经读过 [AI Agent 新手入门路线图](../notes/agent-beginner-learning-path.md)，想直接复制提示词做练习的新手。
- 目标：把“会问 AI”升级成“会给 Agent 一个有边界、有证据、有验收的小任务”。
- 隐私处理：不包含本地路径、账号、聊天记录、后台数据、私有 skill 源码、密钥或未发布项目。

## 为什么单独做这组提示词

新手最常见的问题不是“不会写很长的提示词”，而是任务没有边界：

- 不说清最终交付物。
- 不说哪些资料可以用。
- 不要求来源和自检。
- 不设置停止条件。
- 不区分建议、草稿、真实执行。

OpenAI 的 prompt guidance 强调要围绕目标结果写提示，并为复杂任务设置停止条件和缺证据时的处理方式。OpenAI Agents SDK 把 agent 视为由 instructions、tools、guardrails、handoffs、structured outputs 等组成的系统。Anthropic 的 prompting 文档也强调清晰、直接、具体的指令、上下文和输出格式。

所以这组模板不追求“万能”，只服务一个目标：让新手安全跑通第一个可验证 Agent 小闭环。

## 先记住一个模板公式

每次让 Agent 做事前，先把任务写成这 8 块：

```text
目标：我要你完成什么。
输入：你可以使用哪些资料。
工具：你可以调用哪些能力。
边界：你不能做什么。
输出：最后交付什么格式。
证据：关键结论需要哪些来源。
停止条件：什么时候算完成，什么时候要停下来问我。
自检：交付前你要检查什么。
```

下面 7 个提示词按 7 天入门路线设计。建议顺序使用，不要一上来跳到最后一个完整任务。

## Day 1：判断任务到底需不需要 Agent

用途：避免把所有问题都做成复杂 Agent。

```text
请帮我判断下面这个任务适合用哪种 AI 工作方式。

任务：
{写下你想自动化或交给 AI 的任务}

已知输入：
{资料、网页、文件、表格、截图等}

希望输出：
{笔记、表格、PPT、代码、报告、清单等}

限制：
- 不使用未脱敏隐私数据。
- 不自动发布、付款、下单或修改真实账号设置。
- 不绕过登录、验证码、付费墙或平台限制。

请输出：
1. 它更适合 Chatbot、Workflow、脚本 / CLI、Skill，还是 Agent。
2. 判断理由。
3. 最小可行版本：第一轮只做什么。
4. 需要哪些工具或资料。
5. 主要风险和人工确认点。
6. 如果不适合 Agent，请说明更简单的做法。
```

通过标准：读完后你知道“第一轮到底做什么”，而不是只得到“可以自动化”这种空话。

## Day 2：把一篇公开网页整理成 Markdown 笔记

用途：练习“基于资料回答”，不是凭模型记忆生成。

```text
你是一个 AI Agent 入门学习助手。

任务：阅读下面这篇公开资料，把它整理成一篇适合新手保存到 Markdown 知识库的学习笔记。

资料链接：
{粘贴公开网页链接}

边界：
- 只处理公开可访问内容。
- 不登录账号，不绕过付费墙、验证码或反爬。
- 不复制长篇原文，只做中文转述和结构化整理。
- 资料没有覆盖的内容要标为“不确定 / 待查证”。

输出 Markdown，结构如下：
1. 标题
2. 适合谁
3. 这篇资料解决什么问题
4. 3-5 条核心观点
5. 新手解释
6. 可以立刻实践的一步
7. 不确定 / 待查证
8. 来源链接
9. 自检结论

停止条件：
- 如果页面不可访问、需要登录、需要付费或无法确认来源，请停下来说明原因，不要继续编写正文。
```

通过标准：笔记里有来源、有行动建议、有不确定项，没有长篇搬运。

## Day 3：让 Agent 选择工具前先说明理由

用途：防止 Agent 一上来乱用浏览器、脚本或文件写入。

```text
在执行任务前，请先做工具选择，不要直接开始。

任务：
{任务描述}

可用资料：
{链接、文件、文字说明}

可用工具：
{例如：搜索网页、浏览器、读取文件、写 Markdown、生成 PPT、运行命令}

请先输出一张决策表：

| 候选工具 | 是否需要 | 为什么 | 风险 | 验证方式 |
|---|---|---|---|---|

然后给出：
1. 你推荐的最小工具组合。
2. 哪些工具暂时不要用。
3. 哪些动作需要我确认后才能做。
4. 如果工具失败，你会怎么降级处理。

确认后再执行，不要越过这一步。
```

通过标准：Agent 能解释为什么用工具，也能解释为什么不用某些工具。

## Day 4：自动化浏览前的边界确认

用途：练习安全使用浏览器自动化。

```text
我想让你处理一个网页任务。正式浏览前，请先做边界确认。

网页或目标：
{URL 或目标描述}

任务：
{要提取、比较、整理或检查什么}

请先判断：
1. 这个任务是否只涉及公开资料？
2. 是否可能需要登录、验证码、付费、账号后台或个人数据？
3. 是否有版权、平台规则或反爬风险？
4. 你准备读取哪些页面内容？
5. 你不会做哪些动作？

如果可以继续，请再输出执行计划：
- 打开什么页面。
- 提取什么字段。
- 如何记录来源。
- 如何判断完成。
- 失败时怎么停下来。

没有我的确认，不要提交表单、发布内容、修改账号、下载非公开文件或批量抓取。
```

通过标准：浏览动作前先把权限和风险说清楚。

## Day 5：把长上下文整理成可用输入

用途：不要把所有资料直接塞给 Agent，让它先整理上下文。

```text
请把下面资料整理成适合 Agent 后续执行任务的上下文包。

原始资料：
{粘贴资料、摘要、链接或文件说明}

后续任务：
{下一步要让 Agent 做什么}

请输出：
1. 高信号事实：后续任务必须知道的信息。
2. 可忽略信息：和任务关系不大的内容。
3. 不确定信息：需要继续查证的内容。
4. 术语表：新手可能不懂的词。
5. 推荐给 Agent 的压缩上下文，不超过 500 字。
6. 需要保留的来源链接。

要求：
- 不把不确定内容写成事实。
- 不加入资料里没有的结论。
- 如果资料太杂，请先按主题分组。
```

通过标准：你得到的是一段干净上下文，而不是一堆未经筛选的原文。

## Day 6：给任务加护栏和人工确认点

用途：让 Agent 在高风险动作前停下来。

```text
请为下面这个 Agent 任务设计护栏。

任务：
{任务描述}

可能使用的工具：
{浏览器、文件、命令行、API、PPT、发布平台等}

请输出：
1. 允许做的动作。
2. 禁止做的动作。
3. 必须人工确认的动作。
4. 输入风险：哪些资料不应该直接交给 Agent。
5. 输出风险：哪些内容发布前必须人工审查。
6. 失败时的停止条件。
7. 最小审计清单。

特别注意：
- 不处理未脱敏隐私。
- 不泄露账号、token、cookie、密钥、本地路径。
- 不自动发布、付款、下单或改真实账号设置。
- 不绕过平台限制。
```

通过标准：你能明确告诉 Agent “能做什么、不能做什么、何时必须停”。

## Day 7：完整小闭环执行提示词

用途：把前 6 天能力串起来，完成一次低风险 Agent 任务。

```text
你是一个谨慎的 AI Agent 执行助手。请帮我完成一个低风险学习任务。

任务目标：
把一篇公开资料整理成一篇可保存到知识库的 Markdown 笔记。

资料：
{公开资料链接或正文}

最终交付物：
一篇 Markdown 笔记，包含来源、核心观点、新手解释、行动建议、待查证项和自检结论。

执行规则：
1. 先确认资料是否公开可访问。
2. 再判断是否存在登录、付费、验证码、隐私、版权或平台限制。
3. 如果边界不清，先停下来问我。
4. 只基于可访问资料整理，不编造。
5. 不复制长篇原文。
6. 关键事实保留来源链接。
7. 输出前做一次自检。

输出结构：
# {资料主题} 学习笔记

## 来源
- 标题：
- 链接：
- 阅读日期：

## 适合谁

## 这篇资料解决什么问题

## 核心观点

## 新手解释

## 可以立刻实践的一步

## 不确定 / 待查证

## 我的行动

## 自检结论

停止条件：
- 页面不可访问。
- 需要登录、付费、验证码或绕过限制。
- 来源不可信或无法确认。
- 输出会包含隐私、账号、密钥、本地路径或未授权内容。
```

通过标准：你拿到的是一篇能保存、能复查、能继续迭代的笔记。

## 交付后审计提示词

每次 Agent 给出结果后，用下面这段做最终检查：

```text
请审计你刚才的输出。

检查：
1. 是否完成了原始目标？
2. 哪些内容有来源，哪些是你的解释？
3. 有没有没有证据的判断？
4. 有没有过度复制原文？
5. 有没有隐私、账号、密钥、本地路径或未授权内容？
6. 有没有对新手不友好的术语？
7. 是否给出了可以立刻实践的一步？

请输出：
- 通过项
- 需要修改项
- 修订版
```

## 常见失败样子

| 失败样子 | 应该怎么改 |
|---|---|
| “帮我总结一下这个网页” | 补目标、输出结构、来源要求和停止条件 |
| “你可以自由发挥” | 改成明确受众、格式、边界和验收标准 |
| “用所有工具完成它” | 先让 Agent 做工具选择表 |
| “越详细越好” | 改成只保留和任务相关的高信号内容 |
| “自动发布到平台” | 先生成草稿，人工审核后再发布 |
| “你觉得对就行” | 补自检清单和证据规则 |
