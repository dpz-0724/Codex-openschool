# Codex OpenSchool 传播素材包

这页用于把仓库里已经审计过的内容，转成可以发到小红书、公众号、X、朋友圈或社群的传播素材。

传播目标不是夸大能力，而是让真正需要入门 AI Agent 的读者知道：

- 先读哪一篇。
- 读完能做什么。
- 哪些边界不能碰。
- 为什么这个仓库不是工具链接堆。

仓库链接：

```text
https://github.com/dpz-0724/Codex-openschool
```

## 使用原则

1. 不承诺“全自动赚钱”“替代人类”“一键完成所有工作”。
2. 不展示账号后台、聊天记录、客户材料、密钥、本地路径或未脱敏截图。
3. 不搬运外部文章原文，只发自己的学习路线、模板和方法。
4. 每次传播只讲一个具体问题，不把 17 篇笔记一次性全塞给读者。
5. 发布后记录链接和 Star 变化，判断什么内容真的有效。

## 第二篇：第一个 Agent 实战任务

入口：

- 仓库内页：[第一个 Agent 实战任务](../notes/first-agent-web-research-task.md)
- GitHub 链接：https://github.com/dpz-0724/Codex-openschool/blob/main/notes/first-agent-web-research-task.md

适合读者：

- 已经理解概念，但不知道第一个任务该做什么的人。
- 想练自动化浏览和资料整理，但不想碰账号、隐私和高风险操作的人。
- 想把 AI 回答沉淀成 Markdown 知识库的人。

### 标题备选

1. 第一个 AI Agent 任务别做复杂：先整理一篇公开网页
2. 给新手的 30 分钟 Agent 实战：网页资料变 Markdown 笔记
3. 不碰账号、不碰隐私，怎么安全跑通第一个 Agent 小任务

### 短帖版本

```text
AI Agent 新手的第一个任务，不建议从“操作账号”“自动发布”“批量抓数据”开始。

我建议先做一个低风险闭环：

选一篇公开网页 -> 让 Agent 阅读 -> 生成 Markdown 笔记 -> 自检来源和隐私 -> 保存到知识库

这个练习能同时练到：读资料、标来源、写结构化输出、做边界检查。

实战笔记：
https://github.com/dpz-0724/Codex-openschool/blob/main/notes/first-agent-web-research-task.md
```

### 长帖版本

```text
如果你刚开始学 AI Agent，第一个任务不要选：

- 登录账号
- 自动发帖
- 批量抓数据
- 修改真实项目
- 操作付款或后台

这些任务风险高，失败原因也复杂，新手很难判断到底是提示词问题、工具问题、权限问题，还是网页本身的问题。

更适合的第一个任务是：

把一篇公开网页整理成 Markdown 学习笔记。

它的好处是：

- 来源可验证
- 不需要账号
- 不处理隐私
- 输出容易检查
- 失败也不会影响外部系统

我把这个任务拆成了完整流程：

1. 选择公开资料
2. 让 Agent 先判断页面是否可访问
3. 生成结构化笔记
4. 审计来源、版权、隐私和术语
5. 保存到知识库

实战笔记：
https://github.com/dpz-0724/Codex-openschool/blob/main/notes/first-agent-web-research-task.md

仓库：
https://github.com/dpz-0724/Codex-openschool
```

## 第三篇：Agent 长任务环境规格模板

入口：

- 仓库内页：[Agent 长任务环境规格模板](../notes/agent-long-task-environment-spec.md)
- GitHub 链接：https://github.com/dpz-0724/Codex-openschool/blob/main/notes/agent-long-task-environment-spec.md

适合读者：

- 想把任务交给 Agent 跑 30 分钟以上的人。
- 担心 Agent 越权、跑偏、只口头总结但没有证据的人。
- 已经开始让 Agent 改文件、联网查资料、跑命令的人。

### 标题备选

1. 别直接让 Agent 自主跑：先写一份环境规格
2. 长任务 Agent 为什么容易失控？通常不是模型问题，是环境没圈好
3. 给 Agent 跑 30 分钟以上任务前，我会先填这张表

### 短帖版本

```text
长任务 Agent 最容易出问题的地方，不是“模型不够聪明”，而是环境没有设计好。

开始前至少写清：

- 它能读什么
- 它能改什么
- 哪些动作必须问人
- 每一步留下什么证据
- 最多跑多久
- 什么时候停止
- 失败后怎么回放

我整理了一份可复制的《Agent 长任务环境规格模板》。

链接：
https://github.com/dpz-0724/Codex-openschool/blob/main/notes/agent-long-task-environment-spec.md
```

### 长帖版本

```text
很多人让 Agent 做长任务时，会直接说：

“你自己看着办，持续推进。”

这句话很危险。

真正稳的长任务，不是先问模型够不够强，而是先把环境圈好：

1. 任务边界：它可以自主推进什么，哪些要问人
2. 权限边界：它能读什么、能写什么、禁止什么
3. 产物契约：每一步必须留下什么证据
4. 预算和停止条件：最多跑多久，什么时候停
5. 人在回路：哪些高风险动作必须人工确认
6. 回放和回滚：失败后怎么复现、怎么撤回
7. 隐私和版权：哪些材料不能进入任务环境

我把这套思路整理成了一份可复制模板，适合在让 Agent 跑 30 分钟以上任务前使用。

模板：
https://github.com/dpz-0724/Codex-openschool/blob/main/notes/agent-long-task-environment-spec.md
```

## 第四篇：Agent 代码审查工作流

入口：

- 仓库内页：[Agent 代码审查工作流](../notes/agent-code-review-workflow.md)
- GitHub 链接：https://github.com/dpz-0724/Codex-openschool/blob/main/notes/agent-code-review-workflow.md

适合读者：

- 已经开始让 Agent 写代码、改文件、提交 PR 的人。
- 经常遇到 Agent “说完成了”，但 diff 很难审的人。
- 想把 AI 编程变成可检查流程，而不是一次性生成的人。

### 标题备选

1. 让 Agent 写代码前，先给它加一套审查流程
2. AI 编程别只看结果：我整理了一套 Agent 代码审查闭环
3. 从 Auto-review 到 Bugbot：新手怎么理解 Agent 代码审查

### 短帖版本

```text
让 Agent 写代码，最危险的不是它写错，而是它改了什么、为什么改、有没有越权，你都说不清。

我把近期 Cursor Auto-review 和 Bugbot 的思路，整理成了一篇新手版 Agent 代码审查工作流：

动作前：判断这一步是否越权。
提交前：检查 diff 是否符合任务和安全边界。
PR 后：让自动审查聚焦新增变更，不重复制造噪音。

笔记：
https://github.com/dpz-0724/Codex-openschool/blob/main/notes/agent-code-review-workflow.md
```

### 长帖版本

```text
很多人用 Agent 写代码的流程是：

“帮我实现这个功能。”

然后等 Agent 说完成，再看页面或跑测试。

这个流程的问题是：失败发现得太晚。

真正稳一点的 Agent 编程，应该把审查前移成三段：

1. 动作前：它准备读什么、改什么、跑什么命令，是否越权？
2. 提交前：真实 git diff 是否符合任务，有没有测试和隐私扫描？
3. PR 后：自动审查是否聚焦新增变更，而不是反复制造噪音？

我把 Cursor Auto-review 和 Bugbot 的近期更新，拆成了一篇给新手看的常青笔记。

它不要求你一定用某个工具，而是教你怎么把 Agent 代码生成变成可检查、可回放、可合并的工程流程。

笔记：
https://github.com/dpz-0724/Codex-openschool/blob/main/notes/agent-code-review-workflow.md

仓库：
https://github.com/dpz-0724/Codex-openschool
```
