# Agent 新手低风险练习室

这页把仓库里的路线、笔记和提示词整理成一组可以直接做的小练习。

练习原则：

1. 只使用公开资料、示例文本或你自己写的测试内容。
2. 不登录账号，不自动发布，不批量抓取，不处理未脱敏材料。
3. 每个练习都必须留下一个可检查产物。
4. Agent 说“完成了”不算完成，产物和检查结果才算。

## 练习总览

| # | 练习 | 时间 | 你会练到什么 | 产物 |
|---:|---|---:|---|---|
| 0 | 判断任务是否需要 Agent | 15 分钟 | 区分 Chatbot、Workflow、Script、Agent | 一张任务分类表 |
| 1 | 公开网页整理成 Markdown 笔记 | 30 分钟 | 资料读取、结构化输出、来源自检 | 一篇 `.md` 学习笔记 |
| 2 | 给任务选择工具层 | 30 分钟 | MCP、CLI、API、浏览器自动化选型 | 一张工具选型表 |
| 3 | 给 Agent 任务加权限分级 | 30 分钟 | L0-L4 风险判断和人工确认点 | 一份权限清单 |
| 4 | 写 10 行 Spec 再让 Agent 执行 | 45 分钟 | 把模糊需求变成可验收任务 | 一份 10 行 Spec |
| 5 | 做一次提交前审查 | 45 分钟 | diff 审查、测试缺口、隐私扫描 | 一份审查报告 |
| 6 | 把笔记改成传播短帖 | 30 分钟 | 内容转译、标题、边界表达 | 一段可发布草稿 |

建议顺序：0 -> 1 -> 2 -> 3 -> 4 -> 5 -> 6。

如果你只想先看到结果，从练习 1 开始。

## 练习 0：判断任务是否需要 Agent

适合：完全新手，不确定什么时候该用 Agent。

先读：

- [AI Agent 新手概念 FAQ](../notes/agent-concepts-faq.md)
- [AI Agent 新手术语表](../notes/agent-glossary.md)

输入：

```text
写下 5 个你想交给 AI 做的任务。
```

步骤：

1. 给每个任务标注：Chatbot、Workflow、Script、Agent、人工处理。
2. 写出为什么不是另一个选项。
3. 给每个任务标注风险：低 / 中 / 高。

可复制提示词：

```text
请帮我判断下面这些任务适合用哪种 AI 工作方式。

候选类型：
- Chatbot：回答、解释、改写。
- Workflow：固定步骤、路径清楚。
- Script / CLI：规则明确、可用命令稳定完成。
- Agent：目标明确，但步骤需要边做边判断。
- 人工处理：涉及高风险、隐私、权限或复杂判断。

任务列表：
{粘贴 5 个任务}

请输出表格：
任务 | 推荐方式 | 为什么 | 风险级别 | 不建议怎么做
```

通过标准：

- 至少有 1 个任务被你判断为“不需要 Agent”。
- 每个任务都能说清风险。

常见失败：

- 把所有任务都归为 Agent。
- 只写“更智能”，没有说明为什么。

## 练习 1：公开网页整理成 Markdown 笔记

适合：想马上跑通第一个低风险闭环的人。

先读：

- [第一个 Agent 实战任务](../notes/first-agent-web-research-task.md)
- [Agent 新手任务提示词包](../prompts/agent-beginner-task-prompts.md)

输入：

```text
一篇不需要登录的公开网页。
```

步骤：

1. 选择公开资料，不要选付费墙、登录页、论坛私密帖或个人聊天。
2. 让 Agent 先确认页面是否适合整理。
3. 输出 Markdown 笔记。
4. 检查来源链接、版权、隐私和未验证结论。

可复制提示词：

```text
任务：阅读下面这篇公开资料，把它整理成一篇适合新手保存到 Markdown 知识库的学习笔记。

资料链接：
{公开 URL}

要求：
1. 先判断页面是否公开可读，是否涉及登录、付费墙或隐私材料。
2. 如果不适合整理，请停止并说明原因。
3. 如果适合，输出 Markdown 笔记。
4. 必须保留来源链接。
5. 不要复制长篇原文，只做摘要、结构化整理和学习建议。

输出结构：
# 标题
## 一句话总结
## 关键概念
## 可执行步骤
## 新手容易误解的地方
## 来源
## 仍需核实的问题
```

产物：

- 一篇 Markdown 笔记。
- 一段自检结论。

通过标准：

- 笔记里有来源链接。
- 没有账号、聊天记录、客户材料、本地路径或未授权内容。
- 至少列出 1 个仍需核实的问题。

## 练习 2：给任务选择工具层

适合：已经知道 Agent 可以调工具，但不知道该用什么工具。

先读：

- [Agent 工具调用选型：什么时候用 MCP，什么时候用 CLI](../notes/agent-tooling-mcp-vs-cli.md)

输入：

```text
一个你想让 Agent 做的具体任务。
```

步骤：

1. 判断任务是在读数据、执行动作，还是编排流程。
2. 判断有没有现成 CLI、API、浏览器页面或 MCP。
3. 给出最轻量的方案。
4. 写出不推荐方案。

可复制提示词：

```text
请为下面这个 Agent 任务选择工具层。

任务：
{任务描述}

请判断：
1. 任务主要是 Data、Action 还是 Orchestration？
2. 是否已有 CLI、API、浏览器页面或 MCP？
3. 推荐工具层是什么？
4. 为什么不选其他工具层？
5. 哪些动作必须人工确认？

输出表格：
维度 | 判断 | 理由 | 风险
```

产物：

- 一张工具选型表。

通过标准：

- 推荐方案不是默认“上 MCP”。
- 能说清什么时候用 CLI 或浏览器更合适。
- 有人工确认点。

## 练习 3：给 Agent 任务加权限分级

适合：担心 Agent 越权、乱改文件或直接发布的人。

先读：

- [Agent 自主权限分级清单](../notes/agent-autonomy-permission-ladder.md)

输入：

```text
一个包含工具调用、文件操作或网页操作的任务。
```

步骤：

1. 把任务拆成动作。
2. 给每个动作标 L0-L4。
3. 标出必须人工确认的动作。
4. 写停止条件。

可复制提示词：

```text
请为下面这个 Agent 任务做权限分级。

任务：
{任务描述}

风险级别：
L0：纯解释或规划，不调用工具。
L1：读取公开资料或当前仓库普通文件。
L2：修改本地普通文件、运行检查、生成草稿。
L3：提交、推送、发布、调用外部服务、影响远端状态。
L4：读取密钥、账号、cookie、生产数据、私有聊天、客户材料，默认禁止。

请输出：
动作 | 风险级别 | 是否可自动执行 | 必须人工确认的原因 | 停止条件
```

产物：

- 一份 L0-L4 权限清单。

通过标准：

- L3/L4 动作没有被自动放行。
- 停止条件具体可判断。

## 练习 4：写 10 行 Spec 再让 Agent 执行

适合：经常遇到 Agent “做了但不是我要的”的人。

先读：

- [Agent 编程前的 10 行 Spec 模板](../notes/agent-coding-10-line-spec-template.md)

输入：

```text
一个 30-60 分钟以内的小需求。
```

步骤：

1. 先写 10 行 Spec，不要直接让 Agent 开始。
2. 让 Agent 审计 Spec 是否可验收。
3. 修改到有明确目标、不变量和不做事项。
4. 再让 Agent 执行。

可复制模板：

```text
任务名称：
目标：
用户故事：
输入：
输出：
必须满足的不变量：
明确不做：
可能影响的文件或区域：
验收命令或检查方式：
停止条件：
```

产物：

- 一份 10 行 Spec。
- 一段 Spec 审计结论。

通过标准：

- 有“明确不做”。
- 有验收方式。
- 有停止条件。

## 练习 5：做一次提交前审查

适合：已经让 Agent 改过文件，想知道能不能提交的人。

先读：

- [Agent 代码审查工作流](../notes/agent-code-review-workflow.md)

输入：

```text
一组本地文件改动或一段示例 diff。
```

步骤：

1. 先看真实 diff，不要只看 Agent 总结。
2. 让 Agent 判断是否超范围。
3. 检查测试、链接、隐私和未验证项。
4. 再决定是否提交。

可复制提示词：

```text
请对当前 diff 做提交前审查。

只基于 diff 和命令输出判断，不要基于你记忆里的计划。

请输出：
1. 是否建议提交：是 / 否 / 需要补充验证。
2. P0/P1/P2/P3 问题列表。
3. 是否有超范围改动。
4. 是否有隐私、密钥、本地路径或未授权内容。
5. 已验证项。
6. 未验证项。
7. 建议 commit message。
```

产物：

- 一份审查报告。

通过标准：

- 至少列出“未验证项”。
- 不把“Agent 说完成了”当证据。

## 练习 6：把笔记改成传播短帖

适合：想把学习笔记分享给别人，但不想夸大 Agent 能力的人。

先读：

- [Codex OpenSchool 传播素材包](../share/README.md)

输入：

```text
一篇已经公开、已脱敏的学习笔记。
```

步骤：

1. 只选一个具体问题，不要一次介绍整个仓库。
2. 写 3 个标题。
3. 写短帖版本。
4. 加上原文链接。
5. 检查是否夸大承诺。

可复制提示词：

```text
请把下面这篇公开学习笔记改成一段适合社交平台发布的短帖。

笔记链接：
{公开链接}

要求：
1. 不夸大 Agent 能力。
2. 不承诺全自动、赚钱、替代人类。
3. 只讲一个具体问题。
4. 给 3 个标题备选。
5. 保留原文链接。
6. 说明适合谁读。

输出：
标题备选：
短帖：
适合读者：
边界提醒：
```

产物：

- 3 个标题。
- 一段短帖。
- 一条边界提醒。

通过标准：

- 没有万能承诺。
- 没有未脱敏材料。
- 链接指向公开内容。

## 完成后的复盘

做完任意练习后，用这 5 个问题复盘：

```text
1. 这次任务的输入是否公开、清楚、低风险？
2. Agent 做了哪些动作？有没有越权？
3. 最终产物能否被别人复查？
4. 哪一步最容易失败？
5. 下次是否值得沉淀成提示词、workflow 或 skill？
```

如果第 5 条答案是“是”，再读：

- [Skill 描述怎么写才不误触发](../notes/skill-description-trigger-boundaries.md)
- [Codex 自我进化提示词](../prompts/codex-self-evolution.md)

## 停止线

练习中遇到这些情况，直接停止：

- 需要登录真实账号。
- 需要读取密钥、cookie、`.env`、浏览器登录态或后台数据。
- 输入里有聊天记录、客户材料、订单、联系人或未脱敏截图。
- Agent 准备发布内容、付款、下单、删除远端内容。
- 任务范围从 30 分钟小练习扩大成无法验收的大任务。

先脱敏、缩小范围、补验收标准，再继续。
