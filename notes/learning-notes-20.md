# 第一期开源学习笔记：20 个主题摘要

这些主题来自本地笔记的脱敏整理和公开资料二次归纳。每条都按“适合学什么、可以怎么展开、公开时要删什么”来写，方便后续逐篇扩写。

## Agent 工程化

### 1. 企业 Agent 数据治理落地手册

适合学习企业 Agent 为什么不能只会调用工具，还要解决数据权限、字段口径、审计日志、结果追踪和治理流程。后续可以展开为“数据源接入、权限设计、指标口径、人工审批、审计报告”五部分。公开时只保留通用框架，不放真实数据源、业务字段或客户指标。

### 2. Enterprise Agent Harness：让 Agent 可测、可审计、可上线

已展开为：[Enterprise Agent Harness 学习笔记](enterprise-agent-harness.md)。核心是把 Agent 从 demo 变成可测试、可回放、可上线的工程系统。适合学习 golden cases、trace/replay、guardrails、human approval、release gate 和 audit。

### 3. Agent 操作浏览器和桌面软件的四层技术路线

已展开为：[Agent 自动化浏览与软件操作技术路线](agent-automation-browser.md)。核心是区分 DOM/CDP、视觉点击、系统辅助功能和 RPA/桌面控制。适合学习什么时候该用 Playwright，什么时候需要 Computer Use，什么时候应该直接走 API。

### 4. Vibe Coding / Web Coding 高手学习路径

适合整理成 AI 编程课程路线：从会写提示词，到会读代码、会做小功能、会调试、会验收、会维护。重点不是“让 AI 替你写完”，而是把需求拆解、上下文提供、测试验证和代码审查变成稳定流程。公开时用虚构项目，不放个人 SOP 和真实评审记录。

## Codex 入门与排障

### 5. Codex / Agent 公开视频调研方法

适合整理成内容调研方法论：如何定义样本、提取标题结构、识别高频痛点、归纳选题方向、避免只追热点。公开时只放聚合结论和方法，不放账号 ID、具体链接、非公开数据或采集细节。

### 6. Codex 插件：从 AI 助手到 AI 团队

适合解释 Codex 插件、skills、MCP、apps 和 subagents 的关系。重点是：插件是分发单元，skill 是工作流说明，MCP 是工具连接方式，subagent 是并行协作方式。发布前要复核产品功能是否变化。

### 7. 用 Codex 自动整理成片发布包

适合做内容生产教程：把脚本、标题、封面提示词、发布说明、素材清单、检查项组织成一个可交付文件夹。公开时只用虚构素材名和通用文件结构，不展示真实账号节奏或发布记录。

### 8. Codex 新手最常卡的 3 个问题

适合做新手 FAQ：为什么 Codex 看不到文件、为什么工具不能用、为什么结果和预期不一致。重点是引导用户检查工作目录、权限、上下文、依赖和验收方式。公开时不展示登录状态、令牌、系统截图或本机配置。

### 9. AI 操控浏览器和软件的四层方法

适合做短课脚本：第一层 API，第二层 DOM/CDP，第三层视觉点击，第四层桌面辅助功能。读者能理解“能点”不等于“稳定”，以及为什么优先选结构化接口。公开示例应使用安全 demo 页面。

### 10. Codex Computer Use 用不了？排查顺序

适合整理成 checklist：应用是否可见、窗口是否在前台、权限是否允许、分辨率是否影响点击、是否需要浏览器工具代替桌面工具。公开时不放系统隐私路径、账号状态截图或敏感窗口。

## 办公与内容生产

### 11. Codex 操作 Word 和本地软件的边界

适合复盘“什么时候让 Codex 操作桌面，什么时候改用文档解析库或 Office API”。核心判断是：如果任务要求可重复、可测试、可批量，优先结构化工具；如果只是一次性人工辅助，可以用桌面控制。公开时使用虚构文档。

### 12. 一段提示词让 Codex 做任务复盘

适合和 [Codex 自我进化提示词](../prompts/codex-self-evolution.md) 配套。重点是让 Codex 从历史任务里提炼重复模式，产出可沉淀的 skill、prompt、automation 或 checklist。公开时必须去掉个人任务历史和真实项目上下文。

### 13. Codex API 模式下如何使用 PPT 插件

适合做“API 模式、插件、PPT 工作流”的操作教程。核心是说明输入资料如何组织、如何约束页数和版式、如何验收导出的 PPT。公开时不展示密钥、套餐信息、登录页或真实客户材料。

### 14. PPT Master + Codex 手把手教程

适合做端到端课件制作教程：资料清洗、提纲生成、视觉风格、页面类型、导出检查和人工修订。公开时把示例主题改成虚构或公开知识主题，不放内部材料。

### 15. Codex + PPT Master 的稳定 PPT 流程

适合做工作流对比：直接让模型写 PPT vs 先生成结构化大纲 vs 用 PPT Master 承担页面生成。重点是稳定性来自输入约束、页面模板、素材边界和逐页验收。

## Codex 工作流

### 16. Codex 的 9 个高频能力

适合做 Codex 入门导览：读文件、改代码、跑测试、查文档、操作浏览器、做文档、做 PPT、生成脚本、并行拆任务。公开时避免展示非公开插件入口和个人配置。

### 17. 5 步把 Codex 打造成超级员工

适合做 SOP：给清晰目标、给上下文、让它列计划、让它执行验证、让它复盘沉淀。重点是把 Codex 当“可管理的执行者”，而不是一次性聊天窗口。真实任务全部泛化。

### 18. 给 Codex 做一个任务指挥室

适合整理多线程、多任务、多文件的管理方法：任务看板、状态字段、验收标准、风险等级、交付物清单。公开时只放模板，不放真实项目清单或进度。

### 19. 用 Codex 搭一支专家团队

适合讲 subagent 或多角色协作：研究员、实现者、审稿人、测试员、资料整理员。重点是给每个角色清晰边界，避免多个 agent 重复读同一批材料。公开时只放角色模板和协作规则。

### 20. 多 Agent 避坑指南：为什么协作会浪费 Token

适合复盘多 Agent 的成本和质量问题：任务拆得太碎、上下文重复、缺少验收口径、子任务互相阻塞、结果无人整合。公开时不放真实成本、日志或任务细节，只保留原则和检查表。

## 后续展开顺序

优先展开顺序建议是：企业 Agent 数据治理、Codex 新手排障、PPT Master + Codex、Codex 任务指挥室、多 Agent 避坑指南。它们最适合教学，也最容易脱敏成通用方法。
