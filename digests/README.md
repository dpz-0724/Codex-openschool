3. 至少能追溯到 AIHot 条目；进入主推时优先补一手来源。
4. 不复制长篇原文，不发布未脱敏材料，不把 X 摘要当成最终事实。

## 处理流程

每次处理 AIHot 内容时，按这个顺序：

1. 先拉最近 7 天精选，不默认拉全量。
2. 用关键词筛 Agent、Skill、MCP、Prompt、Automation、Workflow、Guardrail、Eval。
3. 只保留和新手入门、工作流、安全边界、可复用模板相关的条目。
4. 能找到官方博客、GitHub、论文或文档的条目优先进入主推。
5. 只有 X / 社交摘要、缺少稳定入口的条目放入观察中。
6. 最终输出必须包含：为什么重要、新手怎么学、可以落地到仓库的内容。
