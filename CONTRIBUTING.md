# 贡献指南

谢谢你愿意给 Codex OpenSchool 提建议。这个仓库面向 AI Agent 新手，目标是分享可执行、可验证、已脱敏的学习材料，而不是收集所有工具链接。

提交 issue 或 PR 前，请先确认你的内容符合这三个条件：

1. 能解决一个新手真实问题。
2. 有公开来源、操作步骤、判断表或可复制模板。
3. 不包含账号、聊天记录、客户材料、密钥、本地路径、后台截图或未授权内容。

## 可以贡献什么

适合提交：

- 新手能照着做的 Agent 实战任务。
- 已核验的一手资料中文导读。
- 真实可用的 skill / prompt / workflow 推荐。
- 已脱敏的学习笔记、术语解释、避坑清单。
- 链接失效、错别字、目录计数、内部跳转问题。

暂不接收：

- 只有标题、没有正文的方法清单。
- 未验证工具链接合集。
- 需要真实账号、私有数据或后台截图才能讲清楚的案例。
- 外部文章长篇复制或未标注来源的改写。
- 承诺“全自动赚钱”“替代人类”“一键完成所有工作”的宣传内容。

## 内容选题标准

一个好选题至少要回答：

| 问题 | 通过标准 |
|---|---|
| 读者是谁？ | 能说清是新手、教师、内容创作者、产品/运营/开发学习者中的哪类人 |
| 解决什么问题？ | 不是泛泛介绍，而是解决一个具体卡点 |
| 读完能做什么？ | 有步骤、模板、判断表或可检查产物 |
| 来源是什么？ | 有官方博客、GitHub、论文、文档或清楚说明“本地整理” |
| 风险在哪里？ | 写明隐私、版权、账号、权限或自动化边界 |

## 隐私与安全红线

不要提交：

- 本地绝对路径。
- 真实聊天记录。
- 账号后台、数据面板、订单、联系人。
- 客户、公司、内部项目或未发布产品材料。
- API key、token、cookie、`.env`、auth 文件。
- 私有 skill 源码、私有仓库 diff、未脱敏截图。

如果你不确定材料能不能公开，先只提交抽象方法和问题描述，不要贴原始内容。

## 推荐提交方式

### 提内容建议

使用 GitHub issue 模板里的“内容选题建议”。请写清：

- 主题。
- 适合读者。
- 为什么值得收录。
- 公开来源。
- 可能的隐私或版权风险。

维护者会先判断是否进入选题池，不保证所有建议都会写成正文。

### 提链接或错别字

使用“链接 / 错别字 / 小修复”模板。请给出文件路径、位置和建议修改。

### 提 PR

PR 需要满足：

- 新文件已经从相关目录页链接出来。
- README 或专题入口需要同步更新时已经更新。
- 外部来源用链接标注，不复制长篇原文。
- 已完成敏感信息自查。
- 已检查内部链接。

## 推荐文章结构

新增学习笔记建议使用这个结构：

```markdown
# 标题

## 审计信息

- 来源：
- 作者：
- 适合读者：
- 目标：
- 隐私处理：

## 一句话结论

## 为什么重要

## 操作步骤或判断表

## 常见坑

## 可复制模板

## 推荐搭配阅读
```

不是每篇都必须完全一致，但至少要有读者、目标、正文、边界和来源。

## PR 前自检

提交 PR 前请确认：

```text
- [ ] 我没有提交账号、聊天、客户材料、密钥、本地路径或未脱敏截图。
- [ ] 我没有复制外部文章长篇原文。
- [ ] 我说明了内容适合谁、解决什么问题。
- [ ] 我补了来源链接或说明“本地整理”。
- [ ] 我更新了相关目录或 README。
- [ ] 我检查了 Markdown 链接。
```

这个仓库宁可少发，也不发空泛或有隐私风险的内容。
